# 00 重建总路线与学习方法

## 1. 先建立全局脑图

后端目录核心是 server/internal，按责任分层：

1. controller：HTTP 层，收参、调用 service、回包。
2. service：业务层，规则校验、流程编排。
3. repository：数据层，生产为 MySQL/Redis，测试通过 testkit 注入内存夹具。
4. model：领域实体。
5. dto：接口请求与响应结构。
6. middleware：跨路由逻辑（鉴权、限流、日志、trace、recover）。
7. pkg：通用能力（cursor、jwt、errors、response、logger）。
8. router：组装全部依赖并注册路由。

## 2. 真正的调用链

以评论点赞为例：

1. 路由命中 /api/v1/comments/:commentId/like。
2. 先过中间件（TraceID -> Logger -> Recover -> RateLimit -> Auth）。
3. 进入 comment handler。
4. handler 提取 userId 和 commentId，调用 comment service。
5. service 进行参数校验和业务语义转换。
6. service 调用 comment repository（生产实现为 MySQL）。
7. repository 在事务中更新 like 明细与 like_count。
8. service 成功后触发 ranking 缓存失效。
9. handler 返回统一结构 {code, message, data}。

## 3. 从零重写顺序

按这个顺序，你最不容易卡住：

1. 通用包：errors + response + cursor。
2. config 与 main（先跑通 fail-fast 启动与依赖注入）。
3. auth 模块（register/login/refresh/logout）。
4. middleware（auth、optional auth、trace、recover、logger、rate limit）。
5. stall + rating。
6. comment + like + replies。
7. me。
8. ranking + cached repository。
9. migration + mysql repository 完善 + testkit 夹具覆盖测试场景。
10. 完整测试与门禁。

## 4. 每个模块统一开发模板

你自己写模块时，固定用这个模板：

1. 写 model（数据结构）。
2. 写 dto（请求响应）。
3. 写 repository interface（先定义方法，不急着实现）。
4. 写 repository interface + MySQL/Redis 实现。
5. 写 service（业务规则）。
6. 写 handler（HTTP 适配）。
7. 在 router 注册路由。
8. 写 router test 覆盖主链路。
9. 再补 testkit 夹具与 integration test。

## 4.1 重构后新增的架构约束

1. 生产代码不再保留内存仓储 fallback。
2. NewEngine 和 NewEngineWithRepositories 不用于生产装配，测试统一使用显式注入的 NewEngineWithAllRepositories。
3. 测试代码通过 internal/testkit/repositories.go 提供仓储夹具。

## 5. 行级阅读方法（非常重要）

“逐行”不等于逐字翻译。正确做法是：

1. import 块：看依赖关系。
2. type 定义：看领域模型与接口边界。
3. 构造函数：看依赖注入。
4. 核心函数：按 if/for/return 逐分支跟踪。
5. helper 函数：看是否承载关键约束（如 cursor 编解码）。

推荐你读每个函数时都回答三件事：

1. 输入是什么。
2. 约束是什么。
3. 输出和副作用是什么。

## 6. 先掌握的三条硬规则

1. 所有列表接口统一 limit + cursor，不用 offset。
2. 点赞与取消点赞必须幂等。
3. 评论楼中楼规则必须严格校验 rootId/parentId/replyToUserId。

## 7. 重写自检清单

你每写完一个模块，都自查：

1. 是否有统一错误码。
2. 是否有统一响应结构。
3. 是否处理了非法参数。
4. 是否处理了 not found。
5. 是否处理了未登录。
6. 是否有至少一条成功测试和一条失败测试。

## 8. 本手册后续章节怎么用

1. 01 章讲系统入口与装配，你会理解“为什么模块能连起来”。
2. 02-05 章按业务模块深拆，每章都按文件和函数逐行解释。
3. 06 章补齐中间件和 pkg（工程化底座）。
4. 07 章给你完整实操路线，带你从 0 自己敲出可运行项目。

读完 00 章后，你可以开始进入 01 章。
