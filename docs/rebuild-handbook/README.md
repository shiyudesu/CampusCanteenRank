# CampusCanteenRank 后端重建手册（超详细）

## 目标

这套文档面向“只会基础 Go 语法”的学习者，目标是让你可以从 0 开始，逐步重写出当前仓库的后端系统。

文档风格：

1. 按模块拆解。
2. 对关键文件提供逐行级别的讲解（按行号区间解释每一行做了什么）。
3. 每章都提供“你现在应该能写出来什么”。
4. 全程与真实仓库代码对齐。

## 重构对齐说明（2026-03-25）

当前仓库已完成一轮后端重构，阅读本手册时请以以下约束为准：

1. 生产代码中已移除内存仓储实现，仓储接口由 MySQL/Redis 实现承载。
2. 启动策略为 fail-fast：关键依赖缺失或初始化失败会直接启动失败。
3. 路由构造器要求显式注入全部仓储依赖，不再提供默认内存兜底。
4. 测试通过 internal/testkit 提供测试夹具来替代旧内存仓储。

## 学习原则

1. 先能跑，再理解，再优化。
2. 每学完一个模块，立即跑测试并手写一遍。
3. 不要跳过“错误处理”和“边界条件”。这两部分是工程代码和练习代码最大的差别。

## 你将掌握的能力

1. 如何设计分层后端：controller / service / repository / model / dto。
2. 如何实现 JWT 登录态、refresh token 轮换与登出失效。
3. 如何实现游标分页并避免重复和漏数。
4. 如何实现楼中楼评论、点赞幂等、事务一致性。
5. 如何实现排行榜查询与 Redis 缓存装饰器。
6. 如何编写单元测试、路由测试、MySQL 集成测试。
7. 如何在“生产无内存兜底”的前提下设计可测试架构（testkit + 显式注入）。

## 术语速读

1. 游标分页：用 nextCursor 指向下一页起点，不用 offset。
2. 幂等：重复请求不会造成错误的重复副作用。
3. 装饰器仓储：在原仓储外包一层缓存能力，不改原仓储代码。
4. 可选鉴权：有 token 就识别用户，无 token 也允许访问。

## 推荐练习方式

1. 先照文档阅读代码。
2. 第二遍只看标题，尝试自己默写每个函数框架。
3. 第三遍删除你自己的实现后重写。
4. 每次重写后执行：

- go test ./server/...
- go vet ./server/...
- go build ./server/...

如果你按这个节奏完成整套手册，你已经具备把这个项目写进简历并在面试里讲清楚的能力。
