# 07 测试体系与从零复刻实操

本章目标：你不仅能读懂项目，还能自己一把一把写出来，并通过门禁。

---

## 1. 先理解测试分层

本项目测试大致分三层：

1. 单元测试
- 例如 cursor 编解码。

2. 路由测试（偏集成）
- 通过 httptest 直接打路由，验证完整请求链。

3. MySQL 集成测试
- 依赖 MYSQL_DSN，验证真实 SQL/事务行为。

4. testkit 夹具测试
- 通过 internal/testkit 注入内存仓储夹具，覆盖 service/router 行为。

## 3. 你要掌握的门禁命令

每次重构或重写一个模块后都要跑：

1. go test ./server/...
2. go vet ./server/...
3. go build ./server/...

如果只想验证某模块，先局部跑，再全量跑。

## 5. 面试可直接使用的项目讲解框架

你可以按这 6 句讲：

1. 我实现了一个 Gin + MySQL + Redis 的校园食堂评分系统后端。
2. 全部列表统一用 cursor 分页，避免 offset 翻页重复漏数。
3. 评论模块支持楼中楼和点赞幂等，关键写路径使用事务保证一致性。
4. 排行榜支持多维筛选并通过 Redis 装饰器仓储做缓存。
5. 鉴权采用 JWT + refresh token 轮换，并支持设备维度会话隔离。
6. 项目有完整路由测试和集成测试，门禁包含 test、vet、build。
