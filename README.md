# CampusCanteenRank

## 开发规范入口

本仓库开发与协作规范请优先阅读以下文档：

- `agents.md`：全局协作流程、模块化开发与提交规范
- `frontend-dev.md`：前端开发与测试规范
- `backend-dev.md`：后端开发与测试规范
- `api-spec.md`：接口契约基线

代码注释要求：新增或修改代码时必须补充适当注释（解释实现意图、边界条件与关键约束），详细规则见 `agents.md`。

## Go 工作区说明

- 后端模块位于 `server/`，请优先在仓库根目录使用 `go.work` 打开整个工作区，或直接在 `server/` 目录下执行 Go 命令
- 若在 IDE 中出现“包无法导入”，通常是因为只打开了仓库根目录但没有启用工作区模式；当前仓库已提供根目录 `go.work`
- 在仓库根目录运行测试时，请使用 `go test ./server/...`；`go test ./...` 在 workspace 根目录下会因为不包含模块而失败

## 关键流程（摘要）

- 模块化开发：每个模块完成后必须先测试，测试通过后再推送到 GitHub
- 每次开发完成后必须同步更新开发文档（已完成/下一步/待优化/风险），未更新文档视为模块未完成
- 大模型开发如需组件/扩展/SDK：需自行检索官方文档并完成安装，记录来源与版本
- 模型分工强约束：仅前端开发使用 Gemini 3.1 Pro Preview；其余所有任务（无论 agent/mode）统一使用 GPT-5.3 Codex xhigh
- GitHub 提交信息规范（中文）：`feat` / `chore` / `fix` / `refactor` / `docs`

示例：

- `feat: 添加新特性`
- `docs: 更新README`
