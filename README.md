# CampusCanteenRank

## 开发规范入口

本仓库开发与协作规范请优先阅读以下文档：

- `agents.md`：全局协作流程、模块化开发与提交规范
- `frontend-dev.md`：前端开发与测试规范
- `backend-dev.md`：后端开发与测试规范
- `api-spec.md`：接口契约基线

## 关键流程（摘要）

- 模块化开发：每个模块完成后必须先测试，测试通过后再推送到 GitHub
- 大模型开发如需组件/扩展/SDK：需自行检索官方文档并完成安装，记录来源与版本
- GitHub 提交信息规范（中文）：`feat` / `chore` / `fix` / `refactor` / `docs`

示例：

- `feat: 添加新特性`
- `docs: 更新README`
